Установка Kaspersky Endpoint Security

Перед установкой необходимо убедится что в dnf.conf прописаны параметры.

1.1 Kaspersky Endpoint Security – NO GUI
Необходимо перейти в папку «Kes»

dnf install -y kesl-11.2.0-4528.x86_64.rpm


dnf install -y perl-Getopt-Long perl-File-Copy


Останавливаем SELINUX


Запускаем скрипт

/opt/kaspersky/kesl/bin/kesl-setup.pl



Далее установщик Касперского выполнит интерактивную установку. Если будут использоваться параметры антивируса не по умолчанию (например, если используется прокси-сервер), их можно задать в процессе интерактивной установки.

Защита от файловых угроз

kesl-control --start-task 1 -W


Проверка загрузочных секторов

kesl-control --start-task 4 -W



1.2 Kaspersky Endpoint Security – GUI
Необходимо перейти в папку «Kes»

dnf install -y kesl-gui-11.2.0-4528.x86_64.rpm


Перезагружаем

reboot



1.3 Агент Администрирования

dnf -y install klnagent64-11.0.0-29.x86_64.rpm


/opt/kaspersky/klnagent64/lib/bin/setup/postinstall.pl


Команды для взаимодействия с Kaspersky Endpoint Security

Запуск и остановка Kaspersky Endpoint Security Чтобы остановить Kaspersky Endpoint Security в systemd-системе, выполните следующую команду

systemctl stop kesl-supervisor


Чтобы перезапустить Kaspersky Endpoint Security в systemd-системе, выполните следующую команду

systemctl stop kesl-supervisor


Чтобы вывести статус Kaspersky Endpoint Security в systemd-системе, выполните следующую команду

systemctl status kesl-supervisor


ПРОГРАММНЫЙ КОМПЛЕКС ДЛЯ ЗАЩИТЫ РАБОЧИХ МЕСТ VIPNET CLIENT

Необходимо перейти в папку "VipNet"

VipNet - NO GUI

dnf install vipnetclient_ru_x86-64_4.13.1-17762.rpm -y


VipNet-GUI

dnf install vipnetclient-gui_ru_x86-64_4.13.1-17762.rpm -y


Установка DNS 10.14.100.222, 10.17.101.222


Выдаем права для работы "vipnet_dns.sh"

chmod +x vipnet_dns.sh


Запускаем скрипт

./vipnet_dns.sh


Выходим

exit


Убедимся что создался файл в папке VipNet

Проверяем DNS

nano /etc/vipnet.conf



Проверяем изменен ли NssWitch

nano /etc/authselect/nsswitch.conf



Установка ключей:

vipnetclient installkeys <ПУТЬ>


КРИПТОПРОВАЙДЕР CRYPTOPRO CSP 4.0 [MATE]

Необходимо перейти в папку «КриптоПРО --> linux-amd64»

chmod +x install_gui.sh


Запускаем скрипт

./install_gui.sh

dnf install cprocsp-rdr-jacarta*.rpm


Установка средства управления сертификатами token-manager

dnf install token-manager


Установка программного обеспечения для подписи файлов gostcrypto

dnf install gostcryptogui caja-gostcryptogui
